## 四、已完成功能列表（按模块）

### 4.1 页面与界面模块（`index.html` + `style.css`）

1. **页头区域**
   - 展示项目名称与副标题。
   - 固定显示浏览器兼容性说明：仅支持 Chrome / Edge。
   - 非目标浏览器时，兼容性文案高亮为警告样式。

2. **主画布区域**
   - 深色示波器风格全屏 Canvas（`#mainCanvas`），占满可视区主舞台。
   - 支持高 DPI 适配（`devicePixelRatio`，上限 2）。
   - 窗口尺寸变化时自动重算画布大小并重绘。
   - 页面加载遮罩（「正在加载画板…」），初始化完成后自动隐藏。
   - 操作引导文案（未开始涂鸦时显示，提示授权麦克风）。

3. **底部控制栏（4 个按钮）**
   - **开始涂鸦**：发起麦克风授权并启动可视化。
   - **停止涂鸦**：停止动画、关闭麦克风流、释放音频资源。
   - **清空画布**：运行中刷新背景与波形；未运行时清除画布。
   - **保存作品**：界面与逻辑占位，提示「后续里程碑实现」。

4. **状态栏**
   - 实时显示加载、授权、运行、错误等中文反馈。
   - 支持成功 / 错误 / 警告样式区分（`success` / `error` / `warning`）。

5. **按钮交互样式**
   - Hover 高亮、Active 缩放反馈、`focus-visible` 键盘焦点轮廓。
   - 禁用态半透明且不可点击（`cursor: not-allowed`）。

6. **响应式布局**
   - PC 与移动端浏览器基础适配（工具栏换行、页头堆叠等）。

### 4.3 音频核心模块（`AudioEngine`）

| 功能点 | 完成状态 | 实现说明 |
|--------|----------|----------|
| 麦克风权限申请 | ✅ | `navigator.mediaDevices.getUserMedia({ audio: true })` |
| 授权成功获取音频流 | ✅ | `MediaStream` 接入 `MediaStreamSource` |
| 授权失败中文提示 | ✅ | 拒绝、无设备、占用、安全限制等分类文案 |
| `AudioContext` 初始化 | ✅ | 兼容 `webkitAudioContext` |
| `AnalyserNode` 配置 | ✅ | `fftSize = 1024`，`smoothingTimeConstant = 0.65` |
| 时域数据读取 | ✅ | `getTimeDomainData()` → `Uint8Array(1024)` |
| 频域数据读取 | ✅ | `getFrequencyData()` → `Uint8Array(512)`（供后续频谱使用） |
| 平均音量计算 | ✅ | 时域 RMS，归一化 0～1 |
| 主导频率计算 | ✅ | 频域峰值 bin → Hz（供后续粒子/交互使用） |
| 资源释放 | ✅ | `track.stop()`、`disconnect`、`audioContext.close()` |

**授权与启停流程：**

1. 用户点击「开始涂鸦」→ 弹出系统麦克风授权。
2. 允许 → 自动启动可视化主循环。
3. 拒绝或失败 → 状态栏显示明确原因，不进入可视化。
4. 点击「停止涂鸦」→ 停止 `requestAnimationFrame`、释放麦克风与音频上下文。

## 七、技术实现摘要

### 7.1 架构示意

```
index.html
    ├── style.css          … 布局与主题
    └── script.js
            ├── BrowserSupport    … 浏览器检测
            ├── AudioEngine       … Web Audio 采集与分析
            ├── WaveformRenderer  … Canvas 波形绘制
            └── AppUI 主流程      … 按钮、rAF 循环、状态栏
```

### 7.2 关键技术参数

| 参数 | 取值 |
|------|------|
| `AnalyserNode.fftSize` | 1024 |
| 时域缓冲长度 | 1024 |
| 频域缓冲长度 | 512 |
| 波形颜色 | `#39ff14` |
| 波形线宽 | 2px |
| 静音判定阈值（RMS） | 0.012（约 1.2%） |
| 渲染帧循环 | `requestAnimationFrame` |

### 7.3 隐私与数据约束（已实现设计）

- 仅采集麦克风实时音频，用于本地可视化。
- 不上传、不持久化音频数据。
- 停止涂鸦时释放麦克风流与 `AudioContext`。
- 页面引导文案说明「音频仅在本地处理」。

## 九、运行与验证说明

### 9.1 运行方式

1. 使用 **Chrome** 或 **Edge** 打开 `index.html`。  
2. 推荐通过本地服务访问（避免 `file://` 限制麦克风）：  
   ```bash
   python -m http.server 8080
   ```  
   浏览器访问：`http://localhost:8080`

### 9.2 建议验证步骤

1. 打开页面 → 确认加载动画消失、深色画布与四按钮显示正常。  
2. 点击「开始涂鸦」→ 允许麦克风 → 观察荧光绿波形。  
3. 静音 → 波形为水平直线；说话/哼歌 → 波形随振幅变化。  
4. 点击「停止涂鸦」→ 波形停止、麦克风关闭。  
5. 使用 Firefox 等打开 → 确认兼容性警告且按钮不可用。
